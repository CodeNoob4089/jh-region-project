## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 레이드 목록의 팝업 모달이 너무 커서 파티 구성을 한눈에 파악하기 어렵고, 슬롯 1개가 3단 레이아웃(이름/직업/전투력)으로 표시되어 높이를 과도하게 차지함. 빈 자리가 신청자 사이에 섞여 배치되어 가독성이 떨어짐 |
| **Solution** | 모달 너비를 860px로 축소하고, 슬롯을 `[직업 pill] [캐릭터명] [전투력]` 한 줄 행으로 재설계. 빈 자리는 파티 내 맨 아래로 정렬 |
| **UX Effect** | 8인 공대 기준 스크롤 없이 파티 전체 구성을 단번에 확인 가능. 신청 현황 파악 시간 단축 |
| **Core Value** | 레이드 지원 전 파티 밸런스를 빠르게 스캔해 캐릭터 선택 결정을 돕는 핵심 UX 개선 |

## 1. 배경 및 문제 정의

### 1.1 현황

- 모달 최대 너비: `980px`, 최대 높이: `90vh`
- 슬롯 구조 (현재):
  ```
  ┌─────────────────────────┐
  │ 캐릭터명        (18px)  │  ← row 1
  │ [직업pill] [내캐릭터]   │  ← row 2 (meta-row, margin-top 10px)
  │ ┌─ 전투력 박스 ─────┐  │  ← row 3 (margin-top 14px)
  │ │ 전투력   1,620k   │  │
  │ └───────────────────┘  │
  └─────────────────────────┘
  min-height: 96px, padding: 14px
  ```
- 빈 자리: `empty` 클래스 슬롯이 신청자 사이에 위치 순서 그대로 배치됨

### 1.2 목표

- 슬롯 1개를 한 줄(~40px)로 압축
- 모달 너비 860px로 축소
- 빈 자리를 파티 내 항상 맨 아래로 정렬

## 6. 빈 자리 정렬 로직

`buildRaidParties()`는 `slots` 배열을 반환하며 일부 슬롯은 `null`.
RaidModal 렌더링 시 각 파티의 slots를 **신청자 우선, null 뒤로** 정렬:

```js
// 렌더링 직전 (RaidModal.jsx 내부)
const sortedSlots = [...party.slots].sort((a, b) => {
  if (a && !b) return -1;
  if (!a && b) return 1;
  return 0;
});
```
