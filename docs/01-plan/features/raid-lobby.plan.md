## 1. Background

현재 구조:
- `HomePage` (`/`): 카테고리 슬라이더, 어드민 생성 레이드만 표시
- `RaidsPage` (`/raids`): 텍스트 리스트, 어드민 생성 레이드만
- `AdminRaidCreatePage`: 어드민만 접근 가능한 방 생성 폼

요청 목표: 모든 회원이 자유롭게 방을 열고, **직업 아이콘 슬롯이 인라인으로 보이는 로비 UI**에서 참가 신청까지 완결.

## 2. Requirements

### 2-1. 로비 리스트 (`/lobby`)

| # | 요구사항 | 우선순위 |
|---|---|---|
| R1 | 진행 중(is_completed=false)인 모든 레이드를 리스트로 표시 | Must |
| R2 | 페이지 상단 탭: 레이드 종류별 필터 ('심연의재련:루드라' / '침식의정화소') | Must |
| R3 | 카드 상단: 방 이름 / **출발 시간(날짜+시간)** / 인원(X/max) | Must |
| R4 | 카드 본문: 직업 아이콘 슬롯 인라인 (2파티 × 4슬롯) | Must |
| R5 | 카드 우측: 상태에 따라 "참가 신청" / "즉시 가입" / "마감" 버튼 | Must |
| R6 | 내가 신청한 방 카드 하이라이트 (파란 테두리 등) | Must |
| R7 | 시간순 정렬 (기본), 상태순 정렬 전환 가능 | Must |
| R8 | 카드 클릭 시 모달: 파티 상세 구성 + 캐릭터 선택 후 신청/취소 | Must |

### 2-2. 방 생성 (`/lobby/new`)

| # | 요구사항 | 우선순위 |
|---|---|---|
| R9 | 로그인한 모든 회원이 방 생성 가능 | Must |
| R10 | 입력 항목: 레이드 종류(탭 선택), 방 이름(자유 입력), **출발 날짜, 출발 시간**, 최대 인원, 설명 | Must |
| R11 | 생성 후 해당 레이드 탭이 선택된 /lobby로 리다이렉트 | Must |

### 2-3. 방 관리 (수정/삭제)

| # | 요구사항 | 우선순위 |

| R13 | 삭제는 방장 본인만 (Admin 포함) | Must |
| R14 | 수정 페이지는 기존 AdminRaidEditPage 로직 재사용 | Should |

### 2-4. 신청/취소

| # | 요구사항 | 우선순위 |
|---|---|---|
| R15 | 모달 내에서 내 캐릭터 선택 후 신청 | Must |
| R16 | 이미 신청 시 취소 버튼 표시 | Must |
| R17 | 같은 시간대 중복 신청 방지 (기존 로직 유지) | Must |

## 3. DB 변경

```sql
-- raids 테이블에 created_by 컬럼 추가
ALTER TABLE raids
  ADD COLUMN created_by UUID REFERENCES auth.users(id) ON DELETE SET NULL;

-- 기존 방들은 NULL (어드민 방으로 간주)
```

**RLS 정책** (Supabase):
- INSERT: 로그인 사용자라면 누구나
- UPDATE/DELETE: `created_by = auth.uid()` 또는 Admin role

```

│  레이드공대 로비              [+ 방 만들기] [정렬▼] │

│ 방 이름: 트라이엇            4/13(일) 20:00   3/8 │

│                                   [즉시 가입]    │

│  1파티: [●][●][●][●]                             │
│  2파티: [●][●][_][_]                             │
│                                   [참가 신청]    │
├─────────────────────────────────────────────────┤  ← 내 신청 방 (파란 테두리)
│ 방 이름: 천단단250&/호교...   4/13(일) 22:00   1/8 │
│  1파티: [●][_][_][_]                             │
│  2파티: [_][_][_][_]                             │
│                                   [참가 신청]    │

```

**탭**: 레이드 title 값(`심연의재련:루드라`, `침식의정화소`)으로 필터. 탭 전환 시 해당 레이드 방만 표시.  
**출발 시간**: `raid_date` + `start_time` 조합해서 `M/D(요일) HH:mm` 형식으로 카드 상단에 표시.  
**슬롯 아이콘**: 직업 고유 색상 배지 (기존 `JOB_STYLE_MAP` 활용), 빈 슬롯은 회색 점선.  
**방 생성 시 레이드 종류**: 탭 or 드롭다운으로 2종 중 선택 → `title` 컬럼에 저장.

## 5. Risk

| 리스크 | 대응 |
|---|---|
| created_by NULL인 기존 방 수정/삭제 | Admin만 가능하도록 예외 처리 |
| Supabase RLS 미설정 시 보안 취약 | RLS INSERT/UPDATE/DELETE 정책 반드시 설정 |
| 직업 아이콘 에셋 없음 | 기존 JOB_STYLE_MAP 색상 배지로 대체, 추후 이미지 교체 가능 구조 |

## 6. Success Criteria

- [ ] `/lobby` 접근 시 레이드 종류 탭 2개 렌더링, 탭 전환으로 필터링 동작
- [ ] 각 카드에 출발 시간(날짜+시간) 표시
- [ ] 각 카드에 파티 슬롯 인라인 표시 (직업 색상 배지)
- [ ] 로그인 회원이 `/lobby/new`에서 레이드 종류 선택 후 방 생성 가능
- [ ] 카드 클릭 → 모달 → 신청/취소 정상 동작
- [ ] 내 신청 방 하이라이트 표시
- [ ] 시간순 / 상태순 정렬 전환
- [ ] 방장 본인만 자신이 만든 방 삭제 가능
